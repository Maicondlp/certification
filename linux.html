Quiz Preparatório para Certificação em Segurança

Total de 150 perguntas para testar seus conhecimentos. Clique em "Mostrar Resposta" para revelar a solução de cada questão.

Gerenciamento de permissões e controle de acesso

1. Qual comando é usado para alterar as permissões de um arquivo no Linux?

Resposta: `chmod`. Ele pode usar notação octal (ex: 755) ou simbólica (ex: u+x).

2. O que o comando `chown usuario:grupo arquivo.txt` faz?

Resposta: Altera o dono (usuário) e o grupo do arquivo `arquivo.txt`.

3. Para que servem as Listas de Controle de Acesso (ACLs) em sistemas Linux?

Resposta: Para fornecer um mecanismo de permissões mais flexível e granular do que as permissões tradicionais de dono, grupo e outros (ugo).

4. Qual comando visualiza as ACLs de um arquivo?

Resposta: `getfacl nome_do_arquivo`.

5. Como você daria permissão de leitura e escrita para um usuário `ana` em um arquivo `relatorio.doc` sem alterar o dono ou grupo?

Resposta: Usando ACLs com o comando: `setfacl -m u:ana:rw relatorio.doc`.

6. O que significa a permissão `755` em notação octal?

Resposta: Dono tem permissão de leitura, escrita e execução (rwx = 4+2+1=7). Grupo e outros têm permissão de leitura e execução (r-x = 4+0+1=5).

7. O que é o bit SUID (Set User ID)?

Resposta: É um bit de permissão especial que permite que um executável seja executado com os privilégios do dono do arquivo, e não do usuário que o executou.

8. O que é o bit SGID (Set Group ID)?

Resposta: Em um executável, ele roda com os privilégios do grupo do arquivo. Em um diretório, novos arquivos criados herdam o grupo do diretório pai.

9. O que é o "sticky bit" e onde ele é comumente usado?

Resposta: É um bit de permissão que, quando aplicado a um diretório, permite que apenas o dono do arquivo (ou o root) possa renomear ou deletar um arquivo dentro daquele diretório. É comumente usado no diretório `/tmp`.

10. Qual a diferença entre Controle de Acesso Discricionário (DAC) e Controle de Acesso Obrigatório (MAC)?

Resposta: No DAC (padrão Linux com chmod), o dono do objeto define as permissões. No MAC (SELinux/AppArmor), uma política de segurança centralizada define as permissões, sobrepondo-se às decisões do dono.

Configuração de firewalls (iptables, nftables)

11. O que são as três "chains" padrão da tabela `filter` no iptables?

Resposta: `INPUT` (pacotes destinados ao próprio firewall), `OUTPUT` (pacotes gerados pelo firewall) e `FORWARD` (pacotes que atravessam o firewall).

12. Qual comando iptables bloqueia todo o tráfego de entrada do IP `192.168.1.100`?

Resposta: `iptables -A INPUT -s 192.168.1.100 -j DROP`.

13. Qual é a principal vantagem do nftables sobre o iptables?

Resposta: O nftables possui uma sintaxe mais simples e unificada, melhor performance e unifica as ferramentas `iptables`, `ip6tables`, `arptables` e `ebtables` em uma única ferramenta (`nft`).

14. Em nftables, como você criaria uma regra para aceitar conexões SSH (porta 22)?

Resposta: `nft add rule inet filter input tcp dport 22 accept`.

15. O que significa a ação `REJECT` no iptables, e como ela difere de `DROP`?

Resposta: `DROP` descarta o pacote silenciosamente. `REJECT` descarta o pacote, mas envia uma mensagem de erro (ex: "port unreachable") ao remetente.

16. Para que serve a tabela `nat` no iptables?

Resposta: É usada para tradução de endereços de rede (Network Address Translation), principalmente para mascaramento (SNAT) e redirecionamento de portas (DNAT).

17. Qual comando lista todas as regras de iptables com números de linha?

Resposta: `iptables -L -n --line-numbers`.

18. O que é um firewall "stateful"?

Resposta: É um firewall que rastreia o estado das conexões de rede (ex: `NEW`, `ESTABLISHED`, `RELATED`). Ele pode permitir pacotes de resposta a conexões já estabelecidas, mesmo que não haja uma regra explícita para eles.

19. Como você define a política padrão da chain `INPUT` para `DROP` no iptables?

Resposta: `iptables -P INPUT DROP`.

20. No nftables, o que é uma "table family"?

Resposta: Define o tipo de protocolo que a tabela irá processar, como `ip` (para IPv4), `ip6` (para IPv6), `inet` (para ambos IPv4 e IPv6), `arp`, etc.

Análise de tráfego de rede (tcpdump, Wireshark)

21. Qual é a principal diferença entre `tcpdump` e `Wireshark`?

Resposta: `tcpdump` é uma ferramenta de linha de comando, ideal para captura rápida e análise em servidores. `Wireshark` é uma ferramenta gráfica (GUI) que oferece análise visual e muito mais detalhada dos pacotes.

22. Qual comando `tcpdump` captura os 100 primeiros pacotes na interface `eth0`?

Resposta: `tcpdump -i eth0 -c 100`.

23. Como você usaria `tcpdump` para capturar todo o tráfego de e para o host `10.0.0.5`?

Resposta: `tcpdump host 10.0.0.5`.

24. Para que serve a opção `-w` no `tcpdump`?

Resposta: Para escrever (write) a captura de pacotes em um arquivo (formato `.pcap`), em vez de exibi-la na tela. Ex: `tcpdump -w captura.pcap`.

25. O que é um "display filter" no Wireshark?

Resposta: É um filtro aplicado após a captura para exibir apenas os pacotes que correspondem a um critério específico (ex: `ip.addr == 8.8.8.8` ou `tcp.port == 443`), sem descartar os outros pacotes da captura.

26. Qual comando `tcpdump` captura apenas o tráfego na porta 80 ou 443?

Resposta: `tcpdump 'port 80 or port 443'`.

27. No Wireshark, como você pode seguir um stream TCP?

Resposta: Clicando com o botão direito em um pacote TCP da conversa e selecionando "Follow > TCP Stream". Isso remonta e exibe os dados da aplicação trocados na conexão.

28. Para que serve a opção `-n` no `tcpdump`?

Resposta: Para não resolver endereços IP para nomes de host (DNS) e portas para nomes de serviços, o que torna a captura mais rápida.

29. O que é o "three-way handshake" do TCP e como ele aparece no Wireshark?

Resposta: É o processo de estabelecimento de uma conexão TCP. No Wireshark, ele aparece como três pacotes: `[SYN]`, `[SYN, ACK]` e `[ACK]`.

30. Qual filtro de captura (BPF syntax) você usaria no `tcpdump` para ignorar o tráfego SSH?

Resposta: `tcpdump 'not port 22'`.

Gestão segura de repositórios e atualizações

31. Por que é crucial usar HTTPS nos URLs de repositórios de software (como APT ou YUM)?

Resposta: Para criptografar a comunicação entre o cliente e o servidor do repositório, protegendo contra ataques man-in-the-middle (MitM) que poderiam interceptar ou modificar os metadados dos pacotes.

32. Qual é o propósito da assinatura de pacotes com GPG?

Resposta: Garante a autenticidade (confirma que o pacote veio do mantenedor legítimo) e a integridade (confirma que o pacote não foi modificado desde que foi assinado).

33. Em um sistema Debian/Ubuntu, qual arquivo principal define os repositórios do APT?

Resposta: `/etc/apt/sources.list` e os arquivos dentro do diretório `/etc/apt/sources.list.d/`.

34. Em um sistema CentOS/RHEL, onde os arquivos de configuração de repositórios do YUM/DNF são armazenados?

Resposta: No diretório `/etc/yum.repos.d/`.

35. O que acontece se a verificação de assinatura GPG de um pacote falhar durante uma instalação com APT ou YUM?

Resposta: O gerenciador de pacotes irá recusar a instalação do pacote e emitirá um aviso de segurança, pois a autenticidade do pacote não pode ser verificada.

36. O que é o "Princípio do Menor Privilégio" e como ele se aplica à gestão de atualizações?

Resposta: O princípio diz que um processo deve ter apenas as permissões mínimas necessárias para sua função. Em atualizações, isso significa que as tarefas devem ser executadas com privilégios de root apenas quando estritamente necessário.

37. Qual o risco de adicionar um repositório de terceiros não confiável ao seu sistema?

Resposta: O risco principal é a instalação de software malicioso (malware, backdoors) ou pacotes instáveis que podem comprometer a segurança e a estabilidade do sistema.

38. O que o comando `apt-get update` realmente faz?

Resposta: Ele não atualiza os pacotes. Ele resincroniza os arquivos de índice de pacotes a partir de suas fontes (os repositórios), atualizando a lista local de pacotes disponíveis e suas versões.

39. O que é "package pinning" (fixação de pacotes) e quando é útil?

Resposta: É uma técnica para forçar o sistema a instalar uma versão específica de um pacote, mesmo que uma mais nova esteja disponível. É útil para evitar que uma atualização quebre a compatibilidade com outros softwares.

40. Qual a importância de aplicar atualizações de segurança de forma regular e oportuna?

Resposta: Para corrigir vulnerabilidades conhecidas (CVEs) antes que possam ser exploradas por atacantes, reduzindo a superfície de ataque do sistema.

Verificação de integridade com hashes e GPG

41. O que é uma função de hash criptográfica?

Resposta: É um algoritmo que pega uma entrada de tamanho arbitrário e produz uma saída de tamanho fixo (o hash). É projetada para ser uma função de mão única, e qualquer mudança na entrada, por menor que seja, gera um hash completamente diferente.

42. Qual comando gera um hash SHA-256 de um arquivo `documento.pdf`?

Resposta: `sha256sum documento.pdf`.

43. Se você baixar um arquivo e o hash SHA-256 dele corresponder ao publicado no site oficial, o que isso garante?

Resposta: Garante a **integridade** do arquivo, ou seja, que ele não foi corrompido ou modificado durante o download.

44. Por que o MD5 não é mais considerado seguro para verificação de integridade em cenários de alta segurança?

Resposta: Porque ele é vulnerável a "colisões", onde dois arquivos diferentes podem gerar o mesmo hash MD5, permitindo que um atacante substitua um arquivo legítimo por um malicioso com o mesmo hash.

45. O que uma assinatura GPG garante que um hash sozinho não garante?

Resposta: A **autenticidade**. A assinatura GPG confirma quem foi o autor do arquivo (a pessoa que detém a chave privada correspondente à chave pública usada para verificação).

46. Qual comando GPG é usado para verificar a assinatura de um arquivo?

Resposta: `gpg --verify arquivo.asc arquivo_original`. (Onde `arquivo.asc` é o arquivo de assinatura).

47. O que é um "detached signature" (assinatura destacada) no GPG?

Resposta: É uma assinatura que é salva em um arquivo separado (geralmente `.asc` ou `.sig`), em vez de ser incorporada ao arquivo original. Isso mantém o arquivo original intacto.

48. Para verificar uma assinatura GPG, o que você precisa ter em seu "keyring" (chaveiro)?

Resposta: Você precisa ter a chave **pública** da pessoa ou entidade que assinou o arquivo.

49. O que são HMACs (Hash-based Message Authentication Codes)?

Resposta: São códigos de autenticação que usam uma função de hash junto com uma chave secreta. Eles garantem tanto a integridade dos dados quanto a autenticidade, pois apenas quem conhece a chave secreta pode gerar o HMAC correto.

50. Qual o propósito de ferramentas como `AIDE` ou `Tripwire`?

Resposta: São Sistemas de Detecção de Intrusão baseados em Host (HIDS) que monitoram a integridade dos arquivos. Eles criam um banco de dados de hashes de arquivos importantes do sistema e periodicamente os verificam para detectar modificações não autorizadas.

Implementação de criptografia (LUKS, GnuPG)

51. O que é LUKS (Linux Unified Key Setup)?

Resposta: É o padrão para criptografia de disco no Linux. Ele fornece uma camada de criptografia a nível de bloco (block device), permitindo que qualquer partição ou disco inteiro seja criptografado.

52. A criptografia com LUKS protege os dados "em trânsito" ou "em repouso"?

Resposta: Protege os dados **em repouso** (data-at-rest), ou seja, quando estão armazenados no disco rígido ou SSD e o sistema está desligado ou a partição está desmontada.

53. Para que serve o GnuPG (GPG)?

Resposta: É uma implementação do padrão OpenPGP usada para criptografar e assinar dados. É comumente usada para proteger arquivos, e-mails e comunicações.

54. Qual comando GPG você usaria para criptografar um arquivo `segredo.txt` para um destinatário com o e-mail `destinatario@exemplo.com`?

Resposta: `gpg --encrypt --recipient destinatario@exemplo.com segredo.txt`. Isso criará um arquivo `segredo.txt.gpg`.

55. Para descriptografar um arquivo GPG que foi criptografado para você, o que você precisa?

Resposta: Sua **chave privada** e a senha (passphrase) que protege essa chave privada.

56. O que é o "header" de uma partição LUKS e por que ele é importante?

Resposta: O header armazena metadados sobre a criptografia, incluindo os "key slots" que contêm as chaves mestras criptografadas. Se o header for corrompido, os dados se tornam irrecuperáveis.

57. É possível ter múltiplas senhas para descriptografar a mesma partição LUKS?

Resposta: Sim. LUKS suporta até 8 "key slots". Cada slot pode conter a chave mestra criptografada com uma senha ou chave diferente, permitindo acesso por múltiplos usuários ou como backup.

58. Qual é a diferença fundamental entre criptografar um arquivo com GPG e criptografar a partição inteira com LUKS?

Resposta: GPG criptografa objetos individuais (arquivos), enquanto LUKS criptografa um dispositivo de bloco inteiro de forma transparente. Com LUKS, uma vez que a partição é "aberta", o acesso aos arquivos é transparente para o sistema operacional; com GPG, cada arquivo deve ser descriptografado individualmente.

59. O que é a "Web of Trust" (Rede de Confiança) no GPG?

Resposta: É um modelo de confiança descentralizado. Em vez de uma autoridade central, os usuários assinam as chaves públicas uns dos outros para atestar sua validade, criando uma rede de confiança.

60. Qual comando GPG é usado para criptografia simétrica (usando apenas uma senha)?

Resposta: `gpg --symmetric arquivo.txt`. Ele pedirá uma senha para criptografar e a mesma senha para descriptografar.

Hardening com SELinux/AppArmor

61. O que SELinux e AppArmor implementam?

Resposta: Controle de Acesso Obrigatório (MAC - Mandatory Access Control).

62. Qual é a principal diferença na abordagem de políticas entre SELinux e AppArmor?

Resposta: O SELinux usa um sistema de "rotulagem" (labeling) em cada objeto (arquivo, processo, porta), e as políticas são baseadas nesses rótulos. O AppArmor usa perfis baseados em caminhos de arquivos (path-based).

63. Quais são os três modos de operação principais do SELinux?

Resposta: `Enforcing` (Aplica a política e bloqueia violações), `Permissive` (Não bloqueia, mas registra as violações) e `Disabled` (Desativado).

64. Em AppArmor, o que é o modo "complain"?

Resposta: É análogo ao modo `Permissive` do SELinux. O AppArmor não bloqueia as ações que violam a política, mas registra as reclamações (complaints) nos logs do sistema.

65. Por que um processo rodando como `root` pode ser impedido de acessar um arquivo pelo SELinux?

Resposta: Porque no MAC, as regras da política de segurança têm precedência sobre as permissões do usuário, incluindo o `root`. Se a política não permite que o contexto do processo `root` acesse o contexto do arquivo, a ação é bloqueada.

66. Qual comando é usado para verificar o status do SELinux?

Resposta: `sestatus` ou `getenforce`.

67. Onde as violações de política do SELinux são geralmente registradas?

Resposta: No arquivo de log de auditoria, geralmente em `/var/log/audit/audit.log`.

68. O que significa o erro "Permission denied" quando o `chmod 777` não resolve o problema?

Resposta: É um forte indicador de que uma camada de segurança MAC, como SELinux ou AppArmor, está bloqueando o acesso, pois a política MAC se sobrepõe às permissões DAC (chmod).

69. Qual comando é usado para restaurar o contexto de segurança padrão do SELinux para um arquivo?

Resposta: `restorecon -v /caminho/do/arquivo`.

70. Para que servem as "booleanas" do SELinux?

Resposta: São "interruptores" (on/off) que permitem modificar partes da política de segurança do SELinux em tempo de execução, sem a necessidade de recarregar ou recompilar a política inteira.

Aplicação dos princípios CID (Confidencialidade, Integridade, Disponibilidade)

71. Defina Confidencialidade.

Resposta: É a garantia de que a informação é acessível apenas para entidades autorizadas. Previne a divulgação não autorizada de dados.

72. Defina Integridade.

Resposta: É a garantia de que os dados são exatos e completos, e que não foram modificados de forma não autorizada.

73. Defina Disponibilidade.

Resposta: É a garantia de que os sistemas e os dados estão acessíveis e utilizáveis sob demanda por uma entidade autorizada.

74. A criptografia de um disco rígido com LUKS é um controle para qual princípio do CID?

Resposta: Principalmente **Confidencialidade**.

75. A verificação de um arquivo com um hash SHA-256 está relacionada a qual princípio?

Resposta: **Integridade**.

76. A implementação de um cluster de servidores com failover e balanceamento de carga visa garantir qual princípio?

Resposta: **Disponibilidade**.

77. Um ataque de negação de serviço (DDoS) visa comprometer qual princípio do CID?

Resposta: **Disponibilidade**.

78. O uso de senhas fortes e autenticação de dois fatores (2FA) reforça qual princípio?

Resposta: **Confidencialidade**, pois impede o acesso não autorizado.

79. Como as assinaturas digitais (GPG) se relacionam com os princípios CID?

Resposta: Elas garantem **Integridade** (o dado não foi alterado) e **Autenticidade** (que leva ao Não-Repúdio), um conceito relacionado à **Integridade**.

80. A política de backups regulares e testados é um controle para qual princípio?

Resposta: Principalmente **Disponibilidade** (garantindo a recuperação de dados), mas também **Integridade** (restaurando uma versão não corrompida).

Identificação e mitigação de ameaças

81. O que é Phishing?

Resposta: É um tipo de ataque de engenharia social onde um atacante tenta enganar uma vítima para que ela revele informações sensíveis (como credenciais ou dados bancários), geralmente se passando por uma entidade confiável em uma comunicação eletrônica.

82. Qual é a principal mitigação contra ataques de força bruta em senhas?

Resposta: Implementar uma política de bloqueio de conta após um número de tentativas falhas e usar autenticação multifator (MFA).

83. O que é um ataque Man-in-the-Middle (MitM)?

Resposta: É um ataque onde o atacante se posiciona secretamente entre duas partes que acreditam estar se comunicando diretamente, permitindo que ele intercepte, leia e modifique a comunicação.

84. Como o uso de HTTPS mitiga ataques MitM?

Resposta: Criptografando o tráfego entre o cliente e o servidor. Mesmo que o atacante intercepte a comunicação, ele não conseguirá ler o conteúdo, pois não possui a chave de criptografia.

85. O que é Ransomware?

Resposta: É um tipo de malware que criptografa os arquivos da vítima e exige o pagamento de um resgate (ransom) para fornecer a chave de descriptografia.

86. Qual é a defesa mais eficaz contra a perda de dados por Ransomware?

Resposta: Ter backups regulares, testados e armazenados offline ou em um local isolado da rede principal.

87. O que é "hardening" de um sistema?

Resposta: É o processo de proteger um sistema reduzindo sua superfície de ataque. Isso inclui remover softwares desnecessários, desabilitar serviços não utilizados, aplicar patches e configurar o sistema com as melhores práticas de segurança.

88. O que é uma vulnerabilidade de "Zero-Day"?

Resposta: É uma vulnerabilidade em um software que é desconhecida pelo fornecedor ou pelo público. Por não haver uma correção disponível, ela é extremamente perigosa.

89. Como o "Princípio do Menor Privilégio" ajuda a mitigar ameaças?

Resposta: Ao garantir que usuários e processos tenham apenas as permissões mínimas necessárias, ele limita o dano que pode ser causado caso uma conta ou processo seja comprometido.

90. O que é a segmentação de rede e por que é uma boa prática de segurança?

Resposta: É a prática de dividir uma rede em sub-redes menores e isoladas (segmentos ou VLANs). Isso ajuda a conter a propagação de malware e limita o movimento lateral de um atacante caso um segmento seja comprometido.

Avaliação e priorização de riscos

91. O que é um risco em segurança da informação?

Resposta: É a probabilidade de uma ameaça explorar uma vulnerabilidade de um ativo, causando um impacto negativo para a organização. Risco = Ameaça x Vulnerabilidade x Impacto.

92. Qual a diferença entre uma ameaça e uma vulnerabilidade?

Resposta: Uma **ameaça** é qualquer coisa que possa causar dano (ex: um hacker, um malware, um desastre natural). Uma **vulnerabilidade** é uma fraqueza que pode ser explorada por uma ameaça (ex: uma senha fraca, um software desatualizado).

93. O que é avaliação de risco quantitativa?

Resposta: É uma abordagem que tenta atribuir valores monetários objetivos ao risco. Ela calcula métricas como a Expectativa de Perda Anual (ALE).

94. O que é avaliação de risco qualitativa?

Resposta: É uma abordagem mais subjetiva que classifica os riscos usando escalas descritivas, como Baixo, Médio e Alto, com base no julgamento e na experiência dos avaliadores.

95. Quais são as quatro principais estratégias de tratamento de risco?

Resposta: **Mitigar** (aplicar controles), **Transferir** (ex: comprar um seguro), **Aceitar** (quando o custo do controle supera o impacto) e **Evitar** (interromper a atividade que causa o risco).

96. O que é o "apetite a risco" de uma organização?

Resposta: É o nível de risco que uma organização está disposta a aceitar na busca de seus objetivos.

97. Para que serve uma matriz de risco (ou matriz de probabilidade e impacto)?

Resposta: É uma ferramenta visual usada na avaliação de risco qualitativa para ajudar a priorizar os riscos. Ela cruza a probabilidade de ocorrência de um risco com o impacto que ele causaria.

98. O que é um "risco residual"?

Resposta: É o risco que permanece mesmo após a implementação de controles de segurança para mitigá-lo.

99. Na fórmula Risco = Ameaça x Vulnerabilidade, a aplicação de um patch de segurança afeta qual componente?

Resposta: A **Vulnerabilidade**, pois o patch corrige a fraqueza no software.

100. Por que a priorização de riscos é importante?

Resposta: Porque as organizações têm recursos finitos (tempo, dinheiro, pessoal). A priorização garante que os recursos sejam alocados para tratar os riscos mais críticos primeiro.

Conformidade com normas (ISO 27001, GDPR, LGPD)

101. O que é a ISO 27001?

Resposta: É uma norma internacional que especifica os requisitos para estabelecer, implementar, manter e melhorar continuamente um Sistema de Gestão de Segurança da Informação (SGSI).

102. O que é o Anexo A da ISO 27001?

Resposta: É uma lista de referência de objetivos de controle e controles de segurança. Não é obrigatório implementar todos, mas a organização deve justificar por que cada controle foi incluído ou excluído em sua Declaração de Aplicabilidade (SoA).

103. O que é a GDPR (General Data Protection Regulation)?

Resposta: É o regulamento de proteção de dados da União Europeia. Ele estabelece regras rígidas sobre como as organizações devem coletar, usar e proteger os dados pessoais dos cidadãos da UE.

104. O que é a LGPD (Lei Geral de Proteção de Dados)?

Resposta: É a lei brasileira de proteção de dados, fortemente inspirada na GDPR. Ela regula o tratamento de dados pessoais de indivíduos no Brasil.

105. Qual é o conceito de "dado pessoal" segundo a LGPD?

Resposta: É qualquer informação relacionada a uma pessoa natural identificada ou identificável (ex: nome, CPF, e-mail, endereço IP, dados de localização).

106. O que é "Privacy by Design" (Privacidade desde a concepção), um conceito importante na GDPR e LGPD?

Resposta: É a abordagem de incorporar a proteção de dados e a privacidade em todas as fases do desenvolvimento de um sistema ou processo, desde o início, em vez de adicioná-las como um recurso posterior.

107. Qual o papel do DPO (Data Protection Officer) ou Encarregado de Dados?

Resposta: Atuar como o ponto de contato entre a organização, os titulares dos dados e a autoridade nacional de proteção de dados (ANPD no Brasil), além de orientar a organização sobre conformidade.

108. A ISO 27001 é uma lei ou uma norma?

Resposta: É uma **norma**. A conformidade com ela é voluntária, a menos que exigida por contrato ou regulamentação setorial. Já a GDPR e a LGPD são **leis** e sua conformidade é obrigatória.

109. O que é um Relatório de Impacto à Proteção de Dados Pessoais (RIPD)?

Resposta: É um documento exigido pela LGPD que descreve os processos de tratamento de dados pessoais que podem gerar riscos às liberdades civis e aos direitos fundamentais, bem como as medidas para mitigar esses riscos.

110. Cite um direito dos titulares de dados garantido pela LGPD/GDPR.

Resposta: Direito de acesso, direito de retificação, direito de exclusão (direito de ser esquecido), direito à portabilidade dos dados, entre outros.

Criptografia simétrica e assimétrica

111. Qual é a característica fundamental da criptografia simétrica?

Resposta: Ela usa a **mesma chave** para criptografar e descriptografar os dados.

112. Qual é a característica fundamental da criptografia assimétrica?

Resposta: Ela usa um **par de chaves** matematicamente relacionadas: uma chave pública (para criptografar ou verificar assinatura) e uma chave privada (para descriptografar ou assinar).

113. Cite um exemplo de algoritmo de criptografia simétrica.

Resposta: AES (Advanced Encryption Standard), DES, 3DES, Blowfish.

114. Cite um exemplo de algoritmo de criptografia assimétrica.

Resposta: RSA, ECC (Elliptic Curve Cryptography), Diffie-Hellman.

115. Qual é o principal desafio da criptografia simétrica?

Resposta: A distribuição segura da chave. Ambas as partes precisam ter a mesma chave secreta sem que ela seja interceptada por um terceiro.

116. Qual tipo de criptografia é geralmente mais rápido e mais adequado para grandes volumes de dados?

Resposta: Criptografia **simétrica**.

117. Como a criptografia híbrida funciona (como em TLS/SSL)?

Resposta: Ela usa a criptografia **assimétrica** para trocar de forma segura uma chave de sessão. Depois que a chave de sessão é compartilhada, a comunicação passa a ser criptografada com um algoritmo **simétrico** (mais rápido) usando essa chave.

118. Se Alice quer enviar uma mensagem criptografada para Bob usando criptografia assimétrica, qual chave ela usa para criptografar?

Resposta: A chave **pública** de Bob.

119. Se Bob quer criar uma assinatura digital para um documento, qual chave ele usa para assinar?

Resposta: A sua própria chave **privada**.

120. Para que serve o algoritmo Diffie-Hellman?

Resposta: É um protocolo de troca de chaves que permite que duas partes estabeleçam uma chave secreta compartilhada sobre um canal de comunicação inseguro, sem nunca terem se comunicado antes.

Segurança em servidores web (OWASP Top 10)

121. O que é o OWASP Top 10?

Resposta: É um documento de conscientização padrão para desenvolvedores e segurança de aplicações web. Ele representa um amplo consenso sobre os riscos de segurança mais críticos para aplicações web.

122. O que é uma vulnerabilidade de Injeção (Injection) e qual é o tipo mais comum?

Resposta: Ocorre quando dados não confiáveis são enviados para um interpretador como parte de um comando ou consulta. O tipo mais comum é a Injeção de SQL (SQLi).

123. O que é Cross-Site Scripting (XSS)?

Resposta: É uma falha que permite a um atacante injetar scripts maliciosos (geralmente JavaScript) em páginas web visualizadas por outros usuários. O script é executado no navegador da vítima.

124. Qual é a principal defesa contra Injeção de SQL?

Resposta: Usar "prepared statements" (consultas parametrizadas) e ORMs (Object-Relational Mapping), que separam os dados da consulta, impedindo que a entrada do usuário seja interpretada como código SQL.

125. O que é uma vulnerabilidade de "Broken Authentication" (Quebra de Autenticação)?

Resposta: São falhas na implementação das funções de autenticação e gerenciamento de sessão, permitindo que atacantes comprometam senhas, chaves ou tokens de sessão para assumir a identidade de outros usuários.

126. O que é Cross-Site Request Forgery (CSRF ou XSRF)?

Resposta: É um ataque que força um usuário final autenticado a executar ações indesejadas em uma aplicação web na qual ele está atualmente logado. O ataque se aproveita da confiança que a aplicação tem no navegador do usuário.

127. O que é "Security Misconfiguration" (Configuração Incorreta de Segurança)?

Resposta: São falhas de segurança resultantes de configurações inseguras. Exemplos incluem não alterar senhas padrão, habilitar recursos de depuração em produção ou exibir mensagens de erro detalhadas para os usuários.

128. O que é uma vulnerabilidade de "Insecure Deserialization" (Desserialização Insegura)?

Resposta: Ocorre quando dados serializados não confiáveis são processados, podendo levar à execução remota de código ou outros ataques se a aplicação desserializar objetos maliciosos.

129. O que é a categoria "Broken Access Control" (Quebra de Controle de Acesso) do OWASP?

Resposta: Refere-se a falhas na aplicação das restrições de acesso. Isso permite que atacantes acessem funcionalidades ou dados que não deveriam, como acessar a conta de outro usuário ou visualizar páginas de administração.

130. Qual o propósito do cabeçalho HTTP `Content-Security-Policy` (CSP)?

Resposta: É uma camada de segurança que ajuda a detectar e mitigar certos tipos de ataques, incluindo Cross-Site Scripting (XSS) e injeção de dados, ao especificar quais fontes de conteúdo (scripts, imagens, etc.) o navegador tem permissão para carregar.

Sistemas IDS/IPS

131. Qual é a principal diferença entre um IDS (Intrusion Detection System) e um IPS (Intrusion Prevention System)?

Resposta: Um IDS **detecta** atividades maliciosas e gera um alerta. Um IPS **detecta** e tenta **prevenir** ativamente o ataque, bloqueando o tráfego malicioso.

132. O que é um NIDS (Network Intrusion Detection System)?

Resposta: É um IDS que é posicionado em um ponto estratégico da rede para monitorar o tráfego de e para todos os dispositivos na rede.

133. O que é um HIDS (Host-based Intrusion Detection System)?

Resposta: É um IDS que roda em um host individual (um servidor ou estação de trabalho) e monitora a atividade interna daquele sistema, como chamadas de sistema e integridade de arquivos.

134. O que é a detecção baseada em assinaturas (signature-based)?

Resposta: É um método de detecção onde o IDS/IPS procura por padrões específicos (assinaturas) de ataques conhecidos no tráfego de rede ou nos logs do sistema.

135. O que é a detecção baseada em anomalias (anomaly-based)?

Resposta: É um método onde o IDS/IPS primeiro estabelece uma linha de base (baseline) do comportamento normal da rede ou do sistema. Qualquer desvio significativo dessa linha de base é sinalizado como uma possível intrusão.

136. Qual é a principal desvantagem da detecção baseada em assinaturas?

Resposta: Ela não consegue detectar ataques novos ou de dia zero (zero-day), pois ainda não existe uma assinatura para eles.

137. Qual é a principal desvantagem da detecção baseada em anomalias?

Resposta: Ela pode gerar um alto número de "falsos positivos", ou seja, alertar sobre atividades legítimas que desviaram da linha de base normal.

138. Onde um IPS deve ser posicionado na rede?

Resposta: Ele deve ser posicionado "inline" (em linha), de modo que todo o tráfego que ele precisa inspecionar passe através dele, permitindo que ele bloqueie pacotes maliciosos.

139. O que é um falso positivo em um IDS/IPS?

Resposta: É um alerta gerado pelo sistema para uma atividade que na verdade é benigna e não representa uma ameaça real.

140. O que é um falso negativo em um IDS/IPS?

Resposta: É a falha do sistema em detectar e alertar sobre um ataque real que ocorreu. Falsos negativos são geralmente mais perigosos que falsos positivos.

Resposta a incidentes e recuperação

141. Quais são as seis fases do ciclo de vida de resposta a incidentes, segundo o NIST?

Resposta: 1. Preparação; 2. Detecção e Análise; 3. Contenção; 4. Erradicação; 5. Recuperação; 6. Atividades Pós-Incidente (Lições Aprendidas).

142. O que ocorre na fase de **Preparação**?

Resposta: A organização se prepara para lidar com incidentes antes que eles ocorram. Isso inclui criar um plano de resposta, treinar a equipe, e ter as ferramentas necessárias prontas.

143. O que é a fase de **Contenção**?

Resposta: É a fase onde ações são tomadas para limitar o dano e impedir que o incidente se espalhe. Isso pode incluir isolar sistemas da rede ou desabilitar certas funcionalidades.

144. Qual é o objetivo da fase de **Erradicação**?

Resposta: Remover a causa raiz do incidente do ambiente. Isso pode significar eliminar um malware, desativar credenciais comprometidas ou corrigir uma vulnerabilidade.

145. O que é feito na fase de **Recuperação**?

Resposta: Os sistemas afetados são restaurados para a operação normal de forma segura. Isso pode envolver a restauração de backups e o monitoramento cuidadoso para garantir que o incidente não retorne.

146. Por que a fase de **Lições Aprendidas** (Pós-Incidente) é crucial?

Resposta: Para analisar o que aconteceu, como a equipe respondeu, e o que pode ser melhorado. Isso ajuda a fortalecer as defesas e a aprimorar o plano de resposta a incidentes para o futuro.

147. O que é a "Cadeia de Custódia" em forense digital e resposta a incidentes?

Resposta: É a documentação cronológica que mostra quem coletou, manuseou, transferiu e protegeu as evidências digitais. É fundamental para garantir que as evidências sejam admissíveis em um processo legal.

148. Qual é a primeira ação a ser tomada ao se detectar um incidente de segurança em andamento?

Resposta: Seguir o plano de resposta a incidentes. Geralmente, o primeiro passo é confirmar e analisar o incidente (Detecção e Análise) para entender seu escopo antes de passar para a Contenção.

149. O que é um "playbook" de resposta a incidentes?

Resposta: É um guia detalhado, passo a passo, que define as ações específicas a serem tomadas pela equipe de segurança para um tipo particular de incidente (ex: um playbook para ransomware, outro para phishing).

150. Por que é arriscado "contra-atacar" um invasor durante um incidente?

Resposta: Pode ser ilegal, pode escalar o conflito, pode fazer com que o atacante apague seus rastros, e pode expor a organização a responsabilidades legais. A resposta deve focar em contenção e erradicação, não em retaliação.
